# Design System — Storybook

A live, isolated workbench for the **shadcn/ui + Tailwind v4** component library in
this repo. Built for **dev** (browse APIs, copy usage, tweak props live) and
**QA** (verify variants, states, light/dark, and accessibility).

## How it's organised

Components are grouped in the sidebar by category — **Buttons, Forms, Display,
Feedback, Overlay, Navigation, Data, Layout**. Each component has at least a
**Showcase** story; key primitives (Button, Badge, Input, Textarea, Alert, Switch,
Checkbox) also expose a **Playground** with live **Controls**.

## Tips

- **Theme toggle** — flip Light/Dark from the toolbar (top). Tokens come straight
  from `src/app/globals.css` (generated from Figma — `pnpm sync:tokens`).
- **Controls** — change props live on Playground stories.
- **Accessibility** — the **Accessibility** panel runs axe on every story.
- **Docs** — each component has an auto-generated **Docs** tab (autodocs).

## For QA

Every story runs as a real-browser component test via Vitest:

```bash
pnpm test-storybook        # run all story tests (chromium headless)
pnpm test-storybook:watch  # watch mode
```

## Source of truth

Stories reuse the same demos that power the in-app docs site
(`components/docs/demos.tsx`). Showcase stories are generated by
`pnpm gen:stories`; the rich Playground stories live in `stories/*.stories.tsx`.
