## 1. Core Rules

These rules are non-negotiable when writing any UI in this project:

- **Always use semantic tokens** in `className` — never hardcode hex, rgb, or raw Tailwind palette classes
- **Never write** `text-gray-900`, `bg-white`, `border-neutral-200` — use `text-foreground`, `bg-background`, `border` instead
- **Dark mode is automatic** — token-based classes flip automatically via `.dark`; only use `dark:` for one-off exceptions not covered by a token
- **Radius is relative** — use `rounded-lg` (`var(--radius)`) as the base; never hardcode `rounded-[8px]`
- **Token source of truth** — all values come from `variables-export.json`; do not invent token names

```
        ├── FormLabel
        ├── FormControl   ← wraps Input / Select / etc.
        ├── FormDescription
        └── FormMessage   ← validation error
```

Validation via **Zod** + `zodResolver`. Never write inline validation.

### Dialog

```
Dialog
├── DialogTrigger (asChild)
└── DialogContent
    ├── DialogHeader
    │   ├── DialogTitle        ← required for a11y
    │   └── DialogDescription
    ├── [content]
    └── DialogFooter
```

### Sidebar (Dashboard Shell)

```
SidebarProvider
├── Sidebar (side="left" collapsible="icon")
│   ├── SidebarHeader    ← logo / brand
│   ├── SidebarContent
│   │   └── SidebarGroup → SidebarGroupLabel + SidebarMenu
│   │       └── SidebarMenuItem → SidebarMenuButton
│   └── SidebarFooter    ← user profile / logout
└── SidebarInset
    ├── header (h-16 border-b) ← SidebarTrigger + Breadcrumb
    └── main (p-6)
```

### Data Table

```
Table
├── TableHeader → TableRow → TableHead (×n)
└── TableBody  → TableRow → TableCell (×n)
```

For interactive tables: compose with **TanStack Table** + `DataTableToolbar` (search + filters).

### Navigation

| Pattern | Component |
|---|---|
| Top bar | `NavigationMenu` |
| Sidebar shell | `Sidebar` + `SidebarProvider` |
| In-page tabs | `Tabs → TabsList → TabsTrigger → TabsContent` |
| Breadcrumb | `Breadcrumb → BreadcrumbList → BreadcrumbItem → BreadcrumbLink` |
| Dropdown | `DropdownMenu → DropdownMenuTrigger → DropdownMenuContent` |

## 13. Dark Mode

- Toggle `.dark` class on `<html>` — all semantic tokens flip automatically.
- Use `next-themes` in `app/layout.tsx`:

```tsx
import { ThemeProvider } from "@/components/theme-provider"

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body>
        <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
          {children}
        </ThemeProvider>
      </body>
    </html>
  )
}
```

Use `dark:` prefix only for one-off overrides not covered by a semantic token.

## 14. Accessibility Checklist

- [ ] All interactive elements keyboard-navigable
- [ ] Focus ring visible (`focus-visible:ring-2 focus-visible:ring-ring`)
- [ ] Color is not the sole indicator of state
- [ ] All images have `alt`; decorative images use `alt=""`
- [ ] `<label>` linked via `htmlFor` / `id` (or use `FormLabel`)
- [ ] Every `Dialog` has `DialogTitle`
- [ ] Toast / alert messages use `aria-live="polite"`
- [ ] Icon-only buttons have `aria-label`
- [ ] `<Link>` styled as button uses `asChild`
